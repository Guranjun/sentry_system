# 1.消息模块
## 1.消息模块函数组成
### 1.占位函数声明
    占位函数，防止未声明报错
### 2.msg_make
    提供给外部模块生成通用消息格式的消息
### 3.msg_init
    消息模块私有数据初始化
### 4.msg_send
    提供给外部模块发送消息
### 5.msg_receive
    模块内部调用，接收消息
### 6.msg_release
    归还消息内含的外部模块的数据
### 7.msg_cleanup
    回收数据，线程结束时使用
### 8.msg_deliver_thread
    消息线程
### 9.msg_thread_weakup
    当进程结束时调用回收线程
## 2.消息模块相关提示
    由于你要发一个消息走，同时发送模块不管这个消息包含的数据要去往哪，所以数据一定要被接收的模块给copy走，不然会发生数据被覆盖数据错乱的情况。
    同时要发送的消息数据一定是发送模块本地的数据，所以建议在本地维护一个标志位，与被发送数据封装在一个结构体里，表示数据是否被copy完。
    建议这个标志位为count，而非bool型的变量，因为一个模块的本地数据可能转发向很多不同模块。
## 3.消息模块待完善的地方
    目前这个模块是简单的一条信息发向一个模块，如果要发往多个模块，那么就得多次调用msg_send，所以待完善的点是将目的地作为一个队列，指向多个目的地模块，
    同时得维护一个新的变量，表示目的地的个数，以便消息模块处理。
